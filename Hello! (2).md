# Обзор пользовательского интерфейса

Просмотр изменений является важной задачей, и пользовательский интерфейс Gerrit Web предоставляет множество функциональных возможностей для того, чтобы сделать процесс обзора изменений удобным и эффективным. Это руководство по  обзору пользовательского интерфейса, в котором объясняются различные функции и элементы пользовательского интерфейса.

Экран изменений (Change Screen)
===============================

Экран изменения отображает детали одного изменения и предоставляет различные действия с ним.

Блок “Коммит сообщения” (Commit Message Block)
----------------------------------------------
Фокус экрана изменений находится на Commit Message, так как это самая важная информация об изменении. Идентификатор изменения номера и статус изменения отображаются прямо над Commit Message.
Идентификатор изменения номера представляет собой ссылку на изменение, и при клике по ней обновляет экран изменений. Скопировав ссылку, вы получаете постоянную ссылку на изменение.

Статус изменения показывает состояние изменения:

 - `Needs <label>` : 
Это изменение находится в процессе рассмотрения, и для того, чтобы внести изменения, все еще требуется подтверждение. 
 - `Not <label>` : 
Изменение находится на рассмотрении, и право вето на показанный ярлык препятствует внесению изменений. 
 - `Not Current` : 
Текущий набор патчей устарел. Обратите внимание, что некоторые операции, такие как голосование, недоступны в устаревшем наборе патчей, но только в текущем наборе патчей. -
 - `Ready to Submit` :
Изменения одобрены и могут быть внесены.
 - `Merged` : 
Это изменение было успешно объединено с основной ветвью. 
 - `Abandoned` : 
Это изменение было отменено.
 - `Draft` : 
Это черновик, который виден только владельцу изменения, рецензентам, добавленным к изменению, и пользователям, которым назначена глобальная возможность «Просмотреть черновики».

Блок "Данные коммита" (Commit Info Block)
---------------------------------------
Блок "Данные коммита" показывает информацию о коммите текущего просматриваемого набора патчей.

Он отображает автора и коммиттера как ссылки на список изменений этого человека, которые имеют тот же статус, что и текущее просматриваемое изменение.

Идентификатор коммита, родительский коммит и идентификатор изменения ([Change-Id](http://ci.csn.khai.edu/gerrit/Documentation/user-changeid.html)) отображаются с иконкой «копия в буфер обмена», которая позволяет скопировать идентификатор в буфер обмена.

Если настроена программа для работы с Git, такой как gitweb или Gitiles, то также прикрепится ссылка на Git.
Если merge коммит просмотрен, это выделяется иконкой.
Блок "Данные изменения" (Change Info Block)
Блок "Данные изменения" (Change Info Block) содержит детальную информацию об изменениях и предлагает ряд действий по изменениям.
Change Owner:
Владелец изменения отображается ссылкой на список изменений пользователя, у которых такой же статус, как и у просматриваемого изменения.
Reviewers:
Рецензенты изменения отображаются в виде ссылок на аккаунты.
Для каждого рецензента обозначена подсказка о том, за какие метки рецензенту разрешено голосовать.
Чтобы добавить нового рецензента, нужно кликнуть на кнопку Add…​ Ввод текста в всплывающем текстовом поле активирует автоматическое заполнение имен пользователей и групп.
Рецензентов можно удалить из изменения, нажав на значок x в отображении аккаунта рецензента. Удаление рецензента также удаляет текущие голоса рецензента. Удаление голосов регистрируется как сообщение об изменении.
Удаление рецензентов защищено разрешениями:
Пользователи могут всегда удалять себя.
Владелец изменения может удалить любой нулевой или положительный счет.
Пользователи с правом доступа Remove Reviewer, владелец ветки, владелец проекта и администраторы Gerrit могут удалить любого.
Project / Branch / Topic:
Название проекта, для которого было выполнено изменение, отображается как ссылка на default dashboard (панель по умолчанию) проекта. Если в проекте нету панели по умолчанию, ссылка открывает список открытых изменений в проекте.
Нажав на значок настроек справа, вы перейдете к экрану администрирования проекта.
Текущее имя ветви отображается как ссылка на список со всеми изменениями в этой ветви, которые имеют тот же статус, что и текущее просматриваемое изменение.
Если теме было присвоено изменение, оно отображается под веткой. Нажав на значок редактирования, можно установить тему. Для этого требуется право доступа «Редактировать имя темы» (Edit Topic Name). Чтобы задать тему при закрытом изменении, Edit Topic Name должен быть присвоен флаг force.
Submit Strategy:
Submit Strateg используется для сабмита изменения. Стратегия сабмита отображается только для открытых изменений.
Если изменение невозможно объединить из-за конфликтов путей, это выделено жирной красной меткой Cannot Merge.

Actions:

Submit:
Вносит изменения и добавляет его в очередь объединения. Если возможно, изменения объединяются в ветвь.
